Reading generation

Automatically generate readings when adding cards.

What does work, with an accuracy that i am not qualified to judge, is the genertion of reading (pinyin) for hanzi entered. This works quite similar to the creation of Japanese readings with the Japanese support add-on.

Generating readings

The easiest way to get started is to add a note type “Chinese” and use this.

With this note type or another note type that has “Chinese” in the name and has fields called “Hanzi” and “Ruby”, you add the hanzi to the Chinese field, and when you move from that field to the next, the reading is automatically added to the Ruby field. The data source for the readings is the Unicode Hàn (漢) Database (unihan).

The reading added to the ruby field contains both the hanzi and the pinyin, in the format used for the Japanese reading (furigana) rendering, the hanzi as normal text and the pinyin in square brackets. Which part to show can be selected with the standard templates. Use {{furigana:Ruby}} to show the pinyin above the hanzi, {{kana:Ruby}} to show just the pinyin and {{kanji:Ruby}} for just the hanzi.

When there is a field called “Tones”, a string of digits showing the tones used is added to it.

Color

In the add card dialog and the card browser, the reading is shown in black. But during review, the tones readings are shown with color. This is done by adding CSS classes to the readings. The model then defines colors for these classes.

Changing tones

At the moment, changing the tone of a character is somewhat complicated.

To change the tone,

  • Open the edit note dialog,
  • Click in the pinyin field,
  • Click on the more/arrow down icon and on Edit HTML or hit Ctr+Shift+X
  • In the HTML Editor, change both toneN values to the correct toneM value and also change the (diacritic of the) vowel.
  • And click Close
  • You may also want to change the corresponding value in the Tones field.

Changing the colors

The colors are defined in the card template. See the description at Note type.
